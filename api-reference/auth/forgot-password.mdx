# Forgot Password

Initiate a password reset flow by sending OTP to email.

## Endpoint

```
POST /api/v1/auth/forgot-password
```

## Description

Sends a One-Time Password (OTP) to the user's email for password recovery.

## Rate Limiting

- **Limit:** 3 requests per hour
- **Identifier:** Email address

## Request

<ParamField body="email" type="string" required>
  Email address associated with the account
</ParamField>

## Request Body

```json
{
  "email": "john.doe@example.com"
}
```

## Response

### Success (200)

```json
{
  "success": true,
  "message": "OTP sent to your email. Please check your inbox.",
  "data": {
    "expiresIn": 600
  }
}
```

### User Not Found (404)

```json
{
  "success": false,
  "message": "No user found with this email address",
  "code": "USER_NOT_FOUND"
}
```

## Examples

<Tabs>
  <Tab title="cURL">
    ```bash
    curl -X POST http://localhost:3000/api/v1/auth/forgot-password \
      -H "Content-Type: application/json" \
      -d '{"email": "john.doe@example.com"}'
    ```
  </Tab>

  <Tab title="JavaScript">
    ```javascript
    const response = await fetch(
      'http://localhost:3000/api/v1/auth/forgot-password',
      {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({
          email: 'john.doe@example.com'
        })
      }
    );

    const data = await response.json();
    // OTP has been sent to email
    ```
  </Tab>
</Tabs>

## Next Steps

After requesting password reset:
1. User receives OTP via email
2. Call [Reset Password](./reset-password) with OTP and new password
3. Login with new password

## Security Notes

- OTP expires after 10 minutes
- OTP can only be used once
- Rate limited to 3 requests per hour
- Doesn't reveal if email exists (except to user who enters it)
