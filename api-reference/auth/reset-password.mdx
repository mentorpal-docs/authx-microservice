# Reset Password

Complete password reset with OTP verification.

## Endpoint

```
POST /api/v1/auth/reset-password
```

## Description

Resets user password using OTP received via email from forgot-password endpoint.

## Request

<ParamField body="email" type="string" required>
  Email address associated with the account
</ParamField>

<ParamField body="otp" type="string" required>
  One-Time Password sent to email
</ParamField>

<ParamField body="newPassword" type="string" required>
  New password (must meet complexity requirements)
</ParamField>

## Request Body

```json
{
  "email": "john.doe@example.com",
  "otp": "123456",
  "newPassword": "NewSecurePass456!"
}
```

## Response

### Success (200)

```json
{
  "success": true,
  "message": "Password reset successfully. Please login with your new password."
}
```

### Invalid OTP (400)

```json
{
  "success": false,
  "message": "Invalid or expired OTP",
  "code": "INVALID_OTP"
}
```

### OTP Expired (400)

```json
{
  "success": false,
  "message": "OTP has expired. Please request a new one.",
  "code": "OTP_EXPIRED"
}
```

## Examples

<Tabs>
  <Tab title="JavaScript">
    ```javascript
    const response = await fetch(
      'http://localhost:3000/api/v1/auth/reset-password',
      {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({
          email: 'john.doe@example.com',
          otp: '123456',
          newPassword: 'NewSecurePass456!'
        })
      }
    );

    const data = await response.json();
    if (data.success) {
      // Redirect to login
    }
    ```
  </Tab>
</Tabs>

## Security Notes

- OTP is valid for 10 minutes
- Password must meet complexity requirements
- After reset, user must login again
- All active sessions are invalidated after password reset
