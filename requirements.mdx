## Database Requirements

### MongoDB

**Option 1: Local Installation**
```bash
# macOS
brew install mongodb-community

# Ubuntu
sudo apt-get install mongodb

# Start service
brew services start mongodb-community
```

**Option 2: Docker (Recommended)**
```bash
# Run via docker-compose
docker-compose up -d mongodb
```

**Option 3: MongoDB Atlas (Cloud)**
- Managed MongoDB service
- Free tier available (512 MB)
- Recommended for UAT/Production
- [Create Account](https://www.mongodb.com/cloud/atlas)

**Version:** MongoDB 6.0 or higher  
**Minimum Storage:** 512 MB for development

### Redis

**Option 1: Local Installation**
```bash
# macOS
brew install redis

# Ubuntu
sudo apt-get install redis-server

# Start service
brew services start redis
```

**Option 2: Docker (Recommended)
```bash
# Run via docker-compose
docker-compose up -d redis
```

**Option 3: Redis Enterprise (Cloud)**
- Managed Redis service
- Free tier available
- Recommended for UAT/Production
- [Try Redis Enterprise](https://redis.com)

**Version:** Redis 7.0 or higher  
**Minimum Memory:** 256 MB for development

## Email Service Requirements

For sending password reset and notification emails:

### SMTP Configuration

**Gmail (Recommended for Development)**
1. Enable 2-Factor Authentication
2. Generate [App Password](https://myaccount.google.com/apppasswords)
3. Use in `.env`:
```
EMAIL_USER=your-email@gmail.com
EMAIL_PASSWORD=your-app-password
EMAIL_HOST=smtp.gmail.com
EMAIL_PORT=587
```

**SendGrid (Recommended for Production)**
1. Create [SendGrid Account](https://sendgrid.com)
2. Generate API key
3. Use in `.env`:
```
SENDGRID_API_KEY=your-api-key
```

**Other Providers**
- AWS SES
- Mailgun
- Postmark
- SparkPost

## Environment Variables

All configuration is managed via environment variables. See `.env.development` for examples.

### Core Variables

```bash
# Server
NODE_ENV=development
PORT=3000

# Database
MONGODB_URI=mongodb://localhost:27017/authx
MONGODB_POOL_SIZE=10

# Cache
REDIS_URL=redis://localhost:6379

# Security
JWT_SECRET=your-super-secret-key-change-this
JWT_EXPIRY=1h
REFRESH_TOKEN_EXPIRY=1y
BCRYPT_ROUNDS=10

# Email
EMAIL_USER=your-email@gmail.com
EMAIL_PASSWORD=your-password
EMAIL_HOST=smtp.gmail.com
EMAIL_PORT=587

# CORS
CORS_ORIGIN=http://localhost:3000,http://localhost:3001

# Rate Limiting
RATE_LIMIT_WINDOW=15
RATE_LIMIT_MAX_REQUESTS=100

# Logging
LOG_LEVEL=info
LOG_FILE=logs/combined.log
```

## Network Requirements

### Open Ports

| Port | Service | Usage |
|------|---------|-------|
| 3000 | Application | API server |
| 27017 | MongoDB | Database |
| 6379 | Redis | Cache |
| 8081 | Mongo Express | Database UI |
| 587 | SMTP | Email service |

### Firewall Configuration

**Development:** No special configuration needed

**Production:**
- Only expose port 80/443 (HTTP/HTTPS)
- Internal communication (3000, 27017, 6379) on private network
- Restrict database access to application servers only

## Bandwidth Requirements

**Minimum:** 1 Mbps for production  
**Recommended:** 10 Mbps+ for high-traffic scenarios

### Estimated Data Usage
- API Requests: ~1-5 KB per request
- Database Operations: ~10-50 KB per operation
- Log Files: ~100 MB per month

## Security Considerations

### Minimum Security Checklist

- [ ] Use HTTPS in production (SSL/TLS certificate)
- [ ] Store secrets in secure vault (not in code)
- [ ] Enable database authentication
- [ ] Set strong JWT_SECRET
- [ ] Configure CORS properly
- [ ] Enable rate limiting
- [ ] Implement audit logging
- [ ] Regular backups enabled
- [ ] Firewall rules configured
- [ ] Dependencies kept updated
